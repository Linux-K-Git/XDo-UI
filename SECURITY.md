# Security Policy / 安全策略

## Supported Versions / 支持的版本

We actively support the following versions of XDo-UI:
我们积极支持以下 XDo-UI 版本：

| Version | Supported / 支持情况 |
| ------- | ------------------ |
| 2.4.x   | :white_check_mark: |
| 2.2.x   | :不再支持:                |
| < 2.0   | :不再支持:                |

We generally only provide security updates for the latest released version and the current `main` branch.
我们通常只为最新发布的版本和当前的 `main` 分支提供安全更新。

## Reporting a Vulnerability / 报告漏洞

We take security seriously. If you discover a vulnerability in XDo-UI, please report it to us immediately.
我们非常重视安全。如果您发现 XDo-UI 存在漏洞，请立即向我们报告。

### How to Report / 如何报告
You can report security vulnerabilities by opening a **public issue** on GitHub.  
您可以通过在 GitHub 上提交 **公开 Issue** 来报告安全漏洞。

If you need to contact the maintainers, please also do so by opening an issue, and we will respond as soon as possible.  
如需联系维护者，也请通过提交 Issue 的方式，我们会尽快响应。

### What to Include / 需要包含的内容
*   Description of the vulnerability. (漏洞描述)
*   Steps to reproduce. (复现步骤)
*   Potential impact. (潜在影响)

### Response Timeline / 响应时间线
*   We will acknowledge receipt of your report within 48 hours.
    我们将在 48 小时内确认收到您的报告。
*   We will provide a status update within 1 week.
    我们将在 1 周内提供状态更新。
*   We will aim to release a patch as soon as possible.
    我们将尽力尽快发布补丁。

## Security Best Practices for Users / 用户安全最佳实践
*   **Don't run unknown commands**: XDo-UI is a terminal. Be careful when pasting commands from the internet.
    **不要运行未知的命令**：XDo-UI 是一个终端。从互联网上粘贴命令时请务必小心。
*   **Keep Updated**: Always run the latest version of XDo-UI to ensure you have the latest security patches.
    **保持更新**：始终运行最新版本的 XDo-UI 以确保您拥有最新的安全补丁。
